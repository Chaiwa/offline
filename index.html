This document aims to identify the current issues with creating web applications with offline capabilities and detail the use cases and requirements necessary for future offline solutions. The document will go through the current methods for creating offline web apps, the gap in offline behaviour between web and native, user experiences, using offline with installable web applications and other issues such as security and privacy.

Introduction

One of the most obvious differences between native and web applications is the offline capabilities of native applications. Throughout the past few years a number of solutions have been proposed and implemented (AppCache, LocalStorage) . Due to some successes and difficulties with these solutions we are now seeing newer ones come to light (ServiceWorker). Throughout this document we hope to provide the developers of these solutions, and ultimately the implementers, with the information they need to create a workable offline solution for web applications.
